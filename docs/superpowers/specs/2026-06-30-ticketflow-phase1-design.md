# TicketFlow 第一阶段设计

## 目标

第一阶段先建立可运行、可测试、可继续扩展的项目底座，覆盖登录认证、RBAC、部门和用户管理，并为工单、派单、SLA、附件、通知、报表保留清晰模块边界。

## 架构

项目采用模块化单体，后端以包作为模块边界，前端采用单套 Vue3 管理后台。后端模块包括公共基础、系统权限、工单流程、派单规则、SLA、附件、通知消息和统计报表。第一阶段不把所有业务闭环一次性做满，而是先把权限、实体、接口分组和关键领域算法打稳。

## 后端设计

后端使用 Spring Boot 3、MyBatis-Plus 和 MySQL。登录采用自定义 token，Redis 保存 token 与用户 ID 的映射。RBAC 使用用户、角色、菜单、部门、用户角色、角色菜单六类基础表。数据权限使用本人、本部门、本部门及下级、全部数据四种范围。

## 前端设计

前端使用 Vue3、Element Plus、Pinia 和 Vue Router。界面定位为企业内部运维后台，强调信息密度、清晰状态和高频操作效率，不做营销式首页。第一阶段包含登录页、后台框架、工作台、工单列表、规则配置、系统管理和统计看板骨架。

## 测试设计

第一阶段优先覆盖纯领域规则：数据权限判断、工单状态流转、SLA 截止时间计算、自动派单选择。后续阶段再扩展登录鉴权、工单闭环、附件上传、MQ 通知和 Redis 缓存的集成测试。
